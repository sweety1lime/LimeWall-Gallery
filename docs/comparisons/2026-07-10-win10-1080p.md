# Preliminary 720p → 1080p comparison (2026-07-10)

This run validates the comparison pipeline and provides an initial visual check.
It does **not** close the phase 1 quality criterion, which asks for a real 720p
video on a 1440p or 4K monitor.

## Environment

- OS: Windows 10 Home Single Language 22H2, build 19045.6456.
- GPU: NVIDIA GeForce RTX 4060, driver 32.0.16.1047.
- Monitor: `\\.\DISPLAY1`, 1920x1080, 100% DPI.
- Renderer: debug build; libmpv client API 2.5.
- Source: deterministic 1280x720 PNG generated by
  `scripts/generate-upscale-test-chart.ps1`.
- Capture: runtime `screenshot <path>` command, GPU-scaled output, 1920x1080.

## Files and results

| Profile | Screenshot | Size | Observed result |
|---|---|---:|---|
| Eco (bilinear) | [eco-1080p.png](eco-1080p.png) | 1,095,210 B | Softest 1 px grid, small text and diagonal edges. |
| Max (FSR EASU+RCAS) | [max-1080p.png](max-1080p.png) | 1,898,226 B | Finer diagonals and contours remain more defined than Eco; moderate sharpening. |
| Anime4K Mode B Fast | [anime4k-1080p.png](anime4k-1080p.png) | 2,810,849 B | Strongest line reconstruction on cel contours and rings; thin details are emphasized more aggressively. |

All three profiles used the same source frame and target window. Renderer logs
confirmed `FSR upscaling enabled` and `Anime4K enabled: Mode B (Fast)` for the
corresponding captures. Every run stopped cleanly and restored the desktop.

## Live resolution-change test

While the Eco profile was paused, the diagnostic example changed `DISPLAY1`
from 1920x1080 at 165 Hz to 1600x900 at 165 Hz for 12 seconds and then restored
the original mode. The renderer remained alive throughout both transitions.
The watchdog resized the existing surface in both directions:

| State | Evidence | Captured size | SHA-256 |
|---|---|---:|---|
| Temporary mode | [topology-1600x900.png](topology-1600x900.png) | 1600x900 | `D26D96BB4696CC520EB4FFB9F917041A9F57EB0DD40820C0A38B59C09F3BDC97` |
| Restored mode | [topology-restored-1920x1080.png](topology-restored-1920x1080.png) | 1920x1080 | `E4A5DC047A1368EB901EF5F5BF1BD05695E0627CD6E8AAE3542DA9DE4CF0F22F` |

The restored capture is byte-identical to `eco-1080p.png`, which confirms that
the original geometry and rendered output returned exactly. The process then
stopped cleanly and restored the desktop. A physical second-monitor hot-plug
test remains outstanding because only one monitor is connected.
