# Модель безопасности LimeWall

Заметка о защите пользователя от вредоносных обоев. Актуально уже сейчас
(сайдлоад `.wpk` двойным кликом), критично для будущего workshop (фаза 7).

## Главный принцип

**Контейнмент важнее детекции.** Нельзя ревью-модерацией гарантированно поймать
вредонос в произвольном JS, который крутится за рабочим столом 24/7 (обфускация +
подгрузка удалённого кода + отложенный payload обходят любой сканер). Защита
строится на том, что **контент считается враждебным и изолируется**; модерация —
это фильтр риска, а не гарантия.

## Поверхность атаки по типам контента

| Тип | Риск | Вектор |
|-----|------|--------|
| Видео / картинка | низкий | эксплойт декодера ffmpeg/libmpv, decompression-бомба, абсурдное разрешение (DoS) |
| **web (HTML/JS)** | **высокий** | произвольный JS 24/7: майнинг на GPU, эксфильтрация, узел ботнета/DDoS, подгрузка удалённого кода, 0-day WebView2 |
| model3d (glTF) | средний | баги парсера three.js, внешние URI (трекинг/эксфил), гигантские меши (DoS) |
| контейнер `.wpk` (zip) | критично на распаковке | zip-slip / path traversal, zip-бомба, симлинки, абсолютные пути |
| манифест / метаданные | низкий-средний | инъекция при небезопасном рендере в галерее, тайпосквоттинг/импрсонация автора |

## Текущая позиция (что уже сделано)

- **Контейнер `.wpk` защищён** (`crates/wpk`): `ensure_safe_name` режет traversal
  (`..`, `\`, `:`, абсолютные пути, всё кроме `Component::Normal`); `read_capped`/
  `copy_capped` считают фактические байты, а не заголовок (guard от zip-бомб);
  лимиты на число файлов, суммарный и по-файловый размер, размер манифеста;
  ручная распаковка через `File::create` нейтрализует симлинки. Покрыто тестами.
- **Нет нативного моста к странице**: web-поверхность (`crates/platform`, `wry`) не
  регистрирует `with_ipc_handler` — у обоев нет канала к нативному коду.
- **Контент отдаётся через `wallpaper://`, не `file://`** — нет широкого доступа к ФС.
- **Сеть web-обоев заперта по умолчанию** (эта итерация): каждый документ отдаётся
  с CSP-заголовком `WALLPAPER_CSP` — `connect-src` без удалённого хоста убивает
  fetch/XHR/WebSocket наружу, отсутствие remote-origin в `script-src`/`frame-src`
  блокирует удалённый код и remote-iframe; `object-src/base-uri/form-action` заперты;
  `X-Content-Type-Options: nosniff`. Плюс navigation-handler запрещает уход top-frame
  с `wallpaper://`, а new-window-handler запрещает `window.open`. JS исполняется
  свободно — режем **доступ**, а не исполнение. Тест: `served_assets_carry_the_containment_csp`.
- **Информированное согласие перед код-несущими обоями** (эта итерация): web/3D
  `.wpk` и голый `.html` запускают JS на рабочем столе, поэтому не устанавливаются
  молча. UI-импорт (диалог/DnD) инспектирует пакет (`inspect_package`) и для web/3D
  показывает предупреждение до установки; двойной клик по web/3D `.wpk` в бэкенде
  (`import_packages`) не импортирует сразу, а отдаёт UI событие `wpk-consent` для
  подтверждения (fail-safe: пропущенное событие = нет импорта, а не тихий импорт).
  Плейн-медиа импортируется без трения. Тест: `only_code_bearing_packages_require_consent`.
  Оговорка: на самом первом запуске событие консента может уйти до готовности
  webview (тогда пакет просто не импортируется — безопасный дефолт).
- **Watchdog ресурсов** (эта итерация): поток в демоне сэмплит CPU всего дерева
  процессов обоев (рендерер + потомки msedgewebview2) через `platform::resources`
  (нативный Windows-замер, стаб на других ОС). При устойчивом превышении бюджета
  (дефолт: ≥25% всей машины, 5 сэмплов по 4 с ≈ 20 с) демон латч-паузит виновника —
  сперва web-сессии (код-несущие), иначе mpv. Латч (`overbudget`) снимается только
  явным resume пользователя, поэтому runaway-страница не может сама «отпаузиться».
  Гистерезис в `BreachDetector` (fire-once + rearm после спада) исключает флаппинг.
  Тесты: `breach_detector_fires_once_after_sustained_pressure`,
  `resource_guard_pauses_and_names_its_reason`, `cpu_percent_*`, `descendants_*`.
  Двойная польза: это же и есть ядро тезиса «минимальная нагрузка». Пороги —
  консервативный дефолт, требуют живой калибровки (как якорь ≈0% CPU).
- **Причина паузы в UI** (эта итерация): `SessionStatus` несёт `paused_reason`
  (`user|resources|game|battery|lock|display_off`, `#[serde(default)]` для
  version-skew). Демон вычисляет её по прецеденту `pause_reason`; индикатор
  «вежливости» в окне показывает конкретику («высокая нагрузка», «запущена игра»,
  «питание от батареи» и т.д.) вместо немого «На паузе». Тест
  `session_status_reason_round_trips_and_defaults_to_none`.

## Дорожная карта (по убыванию приоритета)

**Фундамент (независим от workshop, нужен уже сейчас):**

1. **Песочница рендер-процесса.** Процесс, крутящий недоверенный контент, — low
   integrity / AppContainer, без записи в ФС вне кэша. Наивысший рычаг после
   сетевого запрета. **Исследовано:** [renderer-sandbox.md](renderer-sandbox.md)
   — полный AppContainer на весь процесс отклонён (UIPI ломает `SetParent` в
   WorkerW); путь — process-mitigation policies сейчас (Фаза 1) + вынос декодера
   libmpv в AppContainer-child через render API позже (Фаза 2); web уже закрыт
   песочницей WebView2.
2. **Badge доверия в UI** (частично закрыто консентом выше): помечать импортированные
   web/3D-обои как «не проверено» до появления подписи/официальной галереи.
3. **Watchdog: GPU-бюджет и настройки.** CPU-бюджет и причина паузы уже есть;
   осталось GPU-мерило (сложнее — нужен PDH/точный источник) и вынос порогов в UI.
4. **Опциональное усиление CSP-хендлером WebResourceRequested** (WebView2 low-level),
   если понадобится блок subresource-загрузок точнее, чем CSP.

**Workshop / галерея (фаза 7):**

5. **Подпись пакетов + revocation list.** Клиент проверяет подпись перед установкой;
   по умолчанию ставим только из официальной галереи; удалённый kill-switch пака.
6. **Серверный ре-энкод видео/картинок** — стирает вредоносную структуру файла;
   туда же ложится платный ИИ-апскейл при импорте.
7. **Автоскан на аплоаде**: строгая валидация структуры `.wpk`, AV/VirusTotal,
   статанализ web/3D (обфускация, сетевые эндпоинты, сигнатуры майнеров, `eval`,
   внешние URI в glTF) → risk score; детонация в инструментированной VM.
8. **Уровни доверия + пост-модерация**: аноним → строгий ревью + сеть заперта;
   verified-паблишеры с бейджем; репорты, телеметрия аномалий, быстрый takedown;
   rate-limit против массовых заливок.

**Сеть по запросу (позже):** если обои легитимно требуют сеть — послаблять
`connect-src` только для флагнутого пакета и с усиленным ревью.

## Фазовый запуск workshop

1. **v1 — только видео/картинки** (серверный ре-энкод + лимиты). Низкий риск.
2. **web/3D-паки — позже**, за строгим ревью + песочница + сеть заперта; до этого —
   sideload-only с предупреждением.

## Честные ограничения

- Полноценная защита стоит денег (инфра ре-энкода, детонация-VM, модераторы,
  подпись, revocation) — это **часть того, почему облачная галерея платная**.
- Отложенный payload может переждать time-limited детонацию; статанализ обходится
  обфускацией — поэтому фундамент — контейнмент (песочница + запрет сети), а не скан.
- Песочница за иконками на Windows нетривиальна (взаимодействие AppContainer с
  WorkerW) — требует отдельного исследования до внедрения.
